## What is PCDiag?

PCDiag is an open-source application that helps diagnose problems on Windows, macOS and Linux systems.

---

## Why PCDiag?

Diagnosing computer problems is often frustrating.

Users are forced to switch between multiple tools to inspect hardware, drivers, event logs, system information, network settings, and software configurations. Even then, finding the actual cause of a problem usually requires technical IT knowledge.

PCDiag aims to bring computer diagnostics into a single platform, helping users identify, understand and resolve problems more efficiently.

---

## Project Goals

- Be lightweight.
- Be cross-platform.
- Work offline by default.
- Respect user privacy.
- Be modular and extensible.
- Provide actionable diagnostics.
- Remain fully open source.

## Roadmap

#### Phase 1 — Foundation

- Define the project architecture.
- Establish coding standards and development guidelines.
- Create the core diagnostics engine.
- Set up CI/CD and automated testing.

#### Phase 2 — Core Diagnostics

- System information.
- Hardware diagnostics.
- Operating system diagnostics.
- Driver diagnostics.
- Storage diagnostics.
- Network diagnostics.

#### Phase 3 — Platform Support

- Windows support.
- macOS support.
- Linux support.
- Platform-specific diagnostics.

#### Phase 4 — Extensibility

- Plugin system.
- Plugin SDK.
- Public plugin API.
- Plugin marketplace (optional).

#### Phase 5 — Reporting

- Human-readable diagnostic reports.
- Export reports.
- Diagnostic history.
- Report comparison.

#### Phase 6 — Advanced Diagnostics

- Automatic issue detection.
- Root cause analysis.
- Troubleshooting recommendations.
- Performance analysis.

#### Phase 7 — Optional AI

- AI-assisted explanations.
- AI-generated troubleshooting suggestions.
- AI-powered report summaries.

AI will always remain optional and never be required to use PCDiag.

#### Phase 8 — Future

- ChromeOS support (if technically feasible).
- Community-developed plugins.
- Additional operating systems.
- Enterprise diagnostics.
