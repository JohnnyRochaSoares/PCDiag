# Technology Stack

## Requirements

The technology stack must:

- Support Windows, macOS and Linux.
- Allow access to system and hardware information.
- Support a modular architecture.
- Be maintainable for many years.
- Have a strong open-source ecosystem.
- Be suitable for a small development team.

---

# Candidates

The following technologies are being considered:

## C++

#### Advantages

- Mature ecosystem.
- Excellent performance.
- Strong support for hardware and operating system integration.
- Large cross-platform GUI ecosystem.

#### Disadvantages

- Higher complexity.
- Manual memory management can introduce risks.

---

## Rust

#### Advantages

- Memory safety guarantees.
- High performance.
- Modern systems programming features.

#### Disadvantages

- Smaller ecosystem compared to C++.

---

## Python + Native Components

#### Advantages

- Fast development.
- Good libraries and tooling.

#### Disadvantages

- Not ideal as the only language for low-level diagnostics.
- May require native extensions.

---

## Go

#### Advantages

- Simple language.
- Good tooling.
- Excellent concurrency support.

#### Disadvantages

- Less suitable for deep hardware integration.
- Smaller ecosystem for desktop applications.
